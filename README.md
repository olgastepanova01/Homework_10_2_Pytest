# Проект "Виджет банковских операций"

## Цель проекта:

Цель проекта "Виджет банковских операций" - реализовать веб-приложение на Python для визуализации банковских операциях клиента, транзакциях.

## Установка и настройка в PyCharm:

1. Клонирование репозитория:
```
git clone https://github.com/username/project-x.git
```

2. Установка зависимостей:
```
pip install -r requirements.txt
```

3. Активация виртуального окружения:
```
python -m venv venv
source venv/bin/activate  # Для Linux/MacOS
venv\Scripts\activate     # Для Windows
```

4. Выполнение миграции посредством команды:
```
python manage.py migrate
```

5. Запуск локального сервера посредством команды:
```
python manage.py runserver
```

## Использование

### Модули
#### Модуль masks
Модуль masks содержит функции `get_mask_card_number` и `get_mask_account` для обработки данных о транзакциях. 
Он находится в директории src/masks.py.
#### Модуль widget
Модуль widget содержит функции `mask_account_card` и `get_date` для обработки данных о транзакциях. 
Он находится в директории src/widget.py.
#### Модуль processing
Модуль processing содержит функции `filter_by_state` и `sort_by_date` для обработки данных о транзакциях. 
Он находится в директории src/processing.py.

### Функции
#### Функция get_mask_card_number
Функция принимает на вход номер карты и возвращает ее маску:
видны первые 6 цифр и последние 4 цифры, остальные символы отображаются звездочками, номер разбит по блокам по 4 цифры, разделенным пробелами.
#### Функция get_mask_account
Функция принимает на вход номер счета и возвращает его маску:
видны только последние 4 цифры номера, а перед ними — две звездочки.
#### Функция mask_account_card
Функция mask_account_card принимает на вход тип и номер карты или номер счета 
и возвращает строку с замаскированным номером.
#### Функция get_date
Функция принимает на вход строку с датой в формате "2024-03-11T02:26:18.671407" 
и возвращает строку с датой в формате "ДД.ММ.ГГГГ".
#### Функция filter_by_state 
Функция принимает список словарей и опционально значение для ключа state (по умолчанию 'EXECUTED'). 
Функция возвращает новый список словарей, содержащий только те словари, у которых ключ state соответствует указанному значению.
#### Функция sort_by_date
Функция принимает список словарей и необязательный параметр, задающий порядок сортировки (по умолчанию — убывание). 
Функция должна возвращать новый список, отсортированный по дате (date).

## Описание пользовательского функционала:

1. Перейдите на страницу Вашего приложения,запустите приложение.
2. Приложение работает успешно, если:
- функция get_mask_card_number принимает на вход номер карты 
и возвращает маску в формате XXXX XX** **** XXXX, где X — это цифра номера;

и возвращает маску в формате **XXXX, где X — это цифра номера;
- функция mask_account_card принимает на вход тип и номер карты или номер счета 
и возвращает строку с замаскированным номером;
- функция get_date принимает на вход строку с датой в формате "2024-03-11T02:26:18.671407" 
и возвращает строку с датой в формате "ДД.ММ.ГГГГ";
- функция filter_by_state принимает список словарей и опционально значение для ключа state (по умолчанию 'EXECUTED')
и возвращает новый список словарей, содержащий только те, у которых ключ соответствует указанному значению;
- функция sort_by_date принимает список словарей и необязательный параметр, задающий порядок сортировки (по умолчанию — убывание) 
и возвращает новый список, отсортированный по дате.

## Тестирование

С целью тестирования проекта используется библиотека 'pytest'.
Конструкиця, применяемая для тестирования различных сценариев - параметризация:
```
@pytest.mark.parametrize
```
Для запуска тестов следует выполнить команду: 
```
pytest
```
Тесты покрывают следующие модули и функции:
- `masks`: функции `get_mask_card_number` и `get_mask_account`.
- `widget`: функции `get_account_card` и `get_date`.
- `processing`: функции `filter_by_state` и `sort_by_date`.

Покрытие тестами составляет более 80% кода проекта.
